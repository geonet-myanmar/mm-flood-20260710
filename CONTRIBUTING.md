# Contributing

Thanks for your interest in improving this flood mapping workflow.

## Development setup

```bash
git clone https://github.com/USERNAME/REPO.git
cd REPO
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

Run the pipeline:

```bash
python flood_inundation_s1.py
```

## How to contribute

1. **Open an issue** for bugs, data-date updates, or method improvements before large changes.
2. **Fork** the repository and create a feature branch (`git checkout -b feature/short-name`).
3. Keep changes focused: prefer small PRs over multi-topic bundles.
4. Update documentation (`README.md`, `docs/METHODOLOGY.md`) when behaviour or parameters change.
5. Do **not** commit large binary caches (e.g. `outputs/vv_db.npz`); they are gitignored and regenerated by the pipeline.
6. Open a pull request with a clear description of *what* changed and *why*.

## Coding guidelines

- Target **Python 3.11+**.
- Prefer readable, documented configuration constants at the top of `flood_inundation_s1.py`.
- Avoid committing API keys or credentials (Planetary Computer STAC access is public and uses signed asset URLs at runtime).
- For scientific parameter changes (thresholds, morphology, resolution), document the rationale in the PR and in `docs/METHODOLOGY.md`.

## Reporting issues

Please include:

- OS and Python version  
- Exact command you ran  
- Full traceback or log excerpt  
- Whether network access to `planetarycomputer.microsoft.com` works  

## Code of conduct

Be respectful and constructive. Harassment or discriminatory behaviour will not be tolerated.
